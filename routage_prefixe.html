Simulateur Pastry : Réseau Dynamique

Routage par préfixe (Base 4, ID sur 4 chiffres)

1. Le Concept : L'entonnoir des identifiants

Imaginons un réseau où les nœuds ont des identifiants de 4 chiffres (en base 4 : 0, 1, 2, 3).
Le but est d'envoyer un message vers la destination 3120.

La règle d'or : À chaque saut, le message doit être envoyé à un nœud qui partage un préfixe commun plus long avec la destination.

  • Le nœud 1023 (préfixe commun : 0 chiffre) cherchera dans sa table un nœud commençant par 3...
  • Le nœud 3211 (préfixe commun : 1 chiffre) cherchera un nœud commençant par 31..
  • Et ainsi de suite jusqu'à arriver à destination !

3. L'Algorithme (Pseudo-code d'un nœud Pastry)

// Exécuté par le nœud courant 'Mon_ID' pour router vers 'Dest_ID'
Fonction RouterParPrefixe(Message, Dest_ID) :
    
    Si (Mon_ID == Dest_ID) Alors
        Accepter(Message) // Le message est arrivé à bon port !
        Retourner

    // 1. Trouver combien de chiffres Mon_ID et Dest_ID ont en commun au début
    L = LongueurPrefixeCommun(Mon_ID, Dest_ID)
    
    // 2. Regarder quel est le chiffre SUIVANT dans l'ID de destination
    Chiffre_Suivant = Dest_ID[L]
    
    // 3. Chercher dans la table de routage
    // La table a des lignes (pour chaque longueur de préfixe L)
    // et des colonnes (pour chaque chiffre possible de 0 à 3)
    Prochain_Noeud = TableRoutage[Ligne: L][Colonne: Chiffre_Suivant]
    
    Si (Prochain_Noeud existe) Alors
        Envoyer(Message, Prochain_Noeud)
    Sinon
        // (Cas d'erreur géré par d'autres mécanismes dans Pastry)
        Renvoyer_Erreur()
