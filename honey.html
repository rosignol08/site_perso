Routage Compact avec SoD

Sense of Direction sur un réseau en Nid d'Abeille (Coordonnées Cubiques)

1. Le Concept : Le réseau s'oriente lui-même

Dans un pavage en nid d'abeille (hexagonal), chaque nœud a 6 voisins. Grâce au Sense of Direction, on attribue au réseau un repère spatial de 3 axes coordonnés (q, r, s) tels que q + r + s = 0. C'est ce qu'on appelle les coordonnées cubiques.

Chaque nœud sait que s'il envoie un message sur son "Port 0", il avance vers l'Est (le long de l'axe q). Le routage ne nécessite plus aucune table en mémoire ! Le nœud compare simplement ses propres coordonnées avec celles de la destination, et active le port qui réduit mathématiquement la distance.

3. L'Algorithme (Pseudo-code)

// Structure de l'adresse d'un nœud (Compact)
Type AdresseNœud = (q, r, s) // tq: q + r + s = 0

// SoD : Liste ordonnée des 6 ports (Vecteurs de direction)
Directions = [
    (+1,  0, -1), // Port 0 : Est
    (+1, -1,  0), // Port 1 : Nord-Est
    ( 0, -1, +1), // Port 2 : Nord-Ouest
    (-1,  0, +1), // Port 3 : Ouest
    (-1, +1,  0), // Port 4 : Sud-Ouest
    ( 0, +1, -1)  // Port 5 : Sud-Est
]

// Exécuté par le nœud Courant C(qc, rc, sc) pour aller vers la Dest T(qt, rt, st)
Fonction RouterMessage(Message, T):
    Si (C == T) Alors
        Accepter(Message) // Arrivé !
        Retourner
    
    // Calcule la distance vers la cible en passant par chaque voisin (O(1))
    MeilleurPort = -1
    MinDist = ∞

    Pour port De 0 A 5:
        Vecteur = Directions[port]
        Voisin_q = C.q + Vecteur.q
        Voisin_r = C.r + Vecteur.r
        Voisin_s = C.s + Vecteur.s
        
        // Distance en coord. cubiques : max(abs(Δq), abs(Δr), abs(Δs))
        Dist = max(abs(Voisin_q - T.q), abs(Voisin_r - T.r), abs(Voisin_s - T.s))
        
        Si (Dist < MinDist) Alors
            MinDist = Dist
            MeilleurPort = port

    // Envoyer physiquement le message sans consulter de table !
    Envoyer(Message, MeilleurPort)
