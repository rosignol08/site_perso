Routage du Plus Court Chemin Distribué

1. Le Principe (Calcul par Phase)

Le but est que chaque nœud connaisse sa distance minimale par rapport à une source (ici S). Au départ, tout le monde pense être à une distance infinie ($\infty$), sauf S qui est à 0.

Quand un nœud trouve un chemin plus court, il met à jour sa valeur et prévient ses voisins. Les voisins recalculent alors leur propre distance. L'algorithme se termine de lui-même quand plus aucun message n'est envoyé.

2. Simulateur Interactif

Topologie : S veut diffuser sa route. Remarquez comment C va d'abord recevoir une mauvaise route via A, avant d'être corrigé par B dans une phase suivante !

3. Pseudo-Code (Logique d'un nœud P_j)

// Variables locales pour le nœud Pj
Entier D_j = ∞  // Distance connue vers la source

// Lors de la réception d'un message (D_i) venant du voisin P_i
Événement RéceptionMessage(D_i, P_i) :
    // On calcule la nouvelle distance potentielle en passant par P_i
    Nouvelle_Dist = D_i + Poids_Lien(P_i, P_j)

    // Si on a trouvé un chemin STRICTEMENT plus court
    Si (Nouvelle_Dist < D_j) Alors
        D_j = Nouvelle_Dist  // Mise à jour (Relâchement)
        
        // On doit prévenir tous nos voisins de notre nouvelle distance !
        Pour chaque voisin P_k de P_j :
            Envoyer le message (D_j) à P_k
    Fin Si
// Si le message ne propose pas de chemin plus court, on l'ignore.
